Each plan is given a version number that consists of a major and a minor version.
The version number is written as major.minor such as version 1.2.

The following describes the concepts and rules associated with plan versioning.

  • When a plan is initially imported, it is assigned a version number of 0.1.

  • When a plan is checked out, the plan is copied and the minor version number is incremented.
    For example, if the current version is 0.1, then the plan is copied to version 0.2 after it is checked out.

  • This new minor version can be deleted using the Delete Version action.

  • Only the latest version of a plan can be checked out or submitted for approval.

  • Once a plan is approved and published, it is promoted to the next major version number.
    For example, if version 0.2 is approved and published, it is promoted to version 1.0.
    Every published plan has a minor version number of zero.

  • A previously published plan that has been unpublished, can be republished with the same version number.
    Essentially, this is to republish a plan that was inadvertently unpublished.

  • Only the highest published version number is shown to guests on the home page.
    It is therefore not necessary to unpublish a plan before publishing an update.
